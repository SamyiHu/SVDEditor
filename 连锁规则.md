## SVDEditor 连锁规则编写指南

连锁操作是 SVDEditor 中的一项功能，可以在删除某个元素时**自动联动删除/修改其他关联元素**。例如删除 GPIO 的 PA0 位域时，自动删除 PBCON 寄存器中的 MODE0 位域。

---

### 一、打开连锁规则编辑界面

菜单栏 → **工具** → **编辑连锁规则...**

也可以通过菜单栏 → **工具** → **启用连锁操作** 来开关全局连锁功能。

---

### 二、连锁规则的数据结构

每条规则由两部分组成：**规则定义（源）** + **连锁动作（目标）**

#### 1. ChainRule（规则定义）

| 字段 | 说明 | 示例 |
|------|------|------|
| `name` | 规则名称 | `"GPIO-端口配置连锁"` |
| `enabled` | 是否启用 | `true` / `false` |
| `source_type` | 源类型 | `"peripheral"` / `"register"` / `"field"` |
| `source_peripheral` | 源外设名（支持通配符 `*`） | `"GPIOA"` / `"GPIO*"` / `"*"` |
| `source_register` | 源寄存器名（支持通配符） | `"MODER"` / `"*"` |
| `source_field` | 源位域名（支持通配符） | `"MODE0"` / `"PA*"` |
| `trigger` | 触发条件 | `"delete"` / `"modify"` |
| `actions` | 连锁动作列表 | 见下方 |

#### 2. ChainAction（连锁动作）

| 字段 | 说明 | 示例 |
|------|------|------|
| `target_peripheral` | 目标外设名（支持通配符和变量） | `"PBCON"` / `"$PERIPHERAL"` |
| `target_register` | 目标寄存器名 | `"MODER"` / `"$REGISTER"` |
| `target_field` | 目标位域名（为空表示整个寄存器） | `"MODE0"` / `"$FIELD"` |
| `action` | 动作类型 | `"delete"` / `"modify"` |
| `description` | 描述信息 | `"同步删除端口配置"` |

---

### 三、通配符和变量替换

#### 通配符（用于匹配源和目标）

| 通配符 | 说明 | 示例 |
|--------|------|------|
| `*` | 匹配所有 | `source_peripheral: "*"` → 匹配任何外设 |
| `前缀*` | 匹配前缀 | `"GPIO*"` → 匹配 GPIOA、GPIOB、GPIOC... |
| `*后缀` | 匹配后缀 | `"*CON"` → 匹配 PBCON、PCCON... |
| `*中间*` | 包含匹配 | `"*OD*"` → 匹配 MODER、ODR... |
| 精确值 | 完全匹配 | `"GPIOA"` → 只匹配 GPIOA |

#### 变量替换（用于动作目标中）

| 变量 | 替换为 | 说明 |
|------|--------|------|
| `$PERIPHERAL` | 源外设名 | 如触发源是 GPIOA，则替换为 `"GPIOA"` |
| `$REGISTER` | 源寄存器名 | 如触发源是 MODER，则替换为 `"MODER"` |
| `$FIELD` | 源位域名 | 如触发源是 MODE0，则替换为 `"MODE0"` |
| `$NAME` | 同 `$PERIPHERAL` | 等价于 `$PERIPHERAL` |

---

### 四、实际示例

#### 示例1：删除 GPIOA 的 PA0 → 同步删除 PBCON.MODE0

```json
{
  "name": "GPIO-端口配置连锁",
  "enabled": true,
  "source_type": "field",
  "source_peripheral": "GPIOA",
  "source_register": "MODER",
  "source_field": "PA0",
  "trigger": "delete",
  "actions": [
    {
      "target_peripheral": "PBCON",
      "target_register": "MODER",
      "target_field": "MODE0",
      "action": "delete",
      "description": "删除GPIOA.PA0时同步删除PBCON.MODE0"
    }
  ]
}
```

#### 示例2：删除任意 GPIO 外设的位域 → 同步删除对应 PBCON 位域

```json
{
  "name": "GPIO通配连锁",
  "enabled": true,
  "source_type": "field",
  "source_peripheral": "GPIO*",
  "source_register": "*",
  "source_field": "*",
  "trigger": "delete",
  "actions": [
    {
      "target_peripheral": "PBCON",
      "target_register": "$REGISTER",
      "target_field": "$FIELD",
      "action": "delete",
      "description": "删除GPIO位域时同步删除PBCON中同名位域"
    }
  ]
}
```

#### 示例3：删除寄存器级联

```json
{
  "name": "删除寄存器级联",
  "enabled": true,
  "source_type": "register",
  "source_peripheral": "GPIOA",
  "source_register": "MODER",
  "source_field": "",
  "trigger": "delete",
  "actions": [
    {
      "target_peripheral": "PBCON",
      "target_register": "MODER",
      "target_field": "",
      "action": "delete",
      "description": "删除GPIOA.MODER时同步删除PBCON.MODER整个寄存器"
    }
  ]
}
```

### 五、在 UI 中编辑动作

在连锁规则编辑对话框中，动作使用文本格式输入，每行一个动作，格式为：

```
目标外设, 目标寄存器, 目标位域, 动作类型
```

例如：
```
PBCON, MODER, MODE0, delete
PCCON, MODER, MODE0, delete
```

- 如果目标位域为空（操作整个寄存器），可以省略或留空：
  ```
  PBCON, MODER, , delete
  ```

### 六、规则文件持久化

规则可以保存为 JSON 文件（通过 `set_rule_file()` 设置路径），格式如下：

```json
{
  "enabled": true,
  "rules": [
    {
      "name": "规则名称",
      "enabled": true,
      "source_type": "field",
      "source_peripheral": "GPIO*",
      "source_register": "*",
      "source_field": "*",
      "trigger": "delete",
      "actions": [
        {
          "target_peripheral": "PBCON",
          "target_register": "$REGISTER",
          "target_field": "$FIELD",
          "action": "delete",
          "description": "说明"
        }
      ],
      "description": "规则描述"
    }
  ]
}
```

### 七、触发时机

当前连锁操作在 `delete_field()` 方法中被触发（第1691-1717行）。当你删除一个位域时：
1. 引擎检查所有已启用的规则
2. 匹配源类型、外设、寄存器、位域和触发条件
3. 执行所有匹配的动作
4. 弹出对话框显示连锁操作结果
