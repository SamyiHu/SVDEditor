# SVDEditor 优化计划

## 概述
本文档记录了SVDEditor的Bug修复和功能优化计划，按优先级排列。

---

## 第一阶段：Bug修复

### 任务1：修复撤销重做不生效 ✅ 已完成
- **问题**：`menu_bar.py` 连接 `self.main_window.undo` / `self.main_window.redo`，但 `MainWindowRefactored` 未定义这两个方法
- **根因**：主窗口缺少 `undo()` / `redo()` 方法；且存在重复的 `command_history` 实例
- **修复方案**：
  1. 在 `MainWindowRefactored` 中添加 `undo()` / `redo()` 方法
  2. 删除重复的 `self.command_history` 
  3. 撤销/重做后刷新UI（树状图、预览、可视化）

### 任务1.1：修复撤销后顺序位置变化 ✅ 已完成
- **问题**：撤销删除操作后，元素位置不在原来的位置
- **根因**：`delete_peripheral`/`delete_register`/`delete_field` 的undo方法将元素追加到末尾而非原位置
- **修复方案**：
  1. `delete_peripheral` undo时重建OrderedDict，按原索引插回
  2. `delete_register` / `delete_field` 同理保持位置
  3. `add_register` / `add_field` / `delete_register` / `delete_field` 全部改为Command模式（原来直接操作不支持撤销）

## 第四阶段：连锁操作

### 任务8：设计连锁规则引擎 ✅ 已完成
- **目标**：支持自定义连锁删除规则（如GPIO PA0删除→PBCON.MODE0删除）
- **涉及文件**：新建 `core/chain_rules.py`
- **具体实现**：
  1. `ChainAction` 数据类：定义连锁动作（目标外设/寄存器/位域、动作类型）
  2. `ChainRule` 数据类：定义连锁规则（源类型、通配符匹配、触发条件）
  3. `ChainRulesEngine` 引擎类：
     - 支持通配符模式匹配（`*`、`GPIO*`）
     - 支持变量替换（`$PERIPHERAL`、`$REGISTER`、`$FIELD`）
     - 支持JSON文件持久化
     - 支持全局开关

  3. 支持添加/删除/编辑规则

### 任务10：连锁选项开关和集成 ✅ 已完成
