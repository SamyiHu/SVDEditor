# 任务6：修复位域图显示继承位域问题（问题2）

## 问题描述
位域图老是显示继承的位域，选中某个非继承的外设，理应是不显示继承的位域的，但现在是显示的，导致位域图很乱。

## 问题分析

### 根本原因
在`bit_field_widget.py`的`set_register`方法中（第29-50行），接受一个`source_peripheral_name`参数，用于标识源外设名称（用于继承外设）。

```python
def set_register(self, register, source_peripheral_name=None):
    """设置寄存器数据
    
    Args:
        register: 寄存器对象
        source_peripheral_name: 源外设名称（用于继承外设）
    """
    self.register = register
    # 更新源外设名称
    if source_peripheral_name is not None:
        if source_peripheral_name != "":
            self.source_peripheral_name = source_peripheral_name
        else:
            # 如果传递了空字符串，清除源外设名称
            self.source_peripheral_name = None
```

**问题所在**：
- 如果没有传递`source_peripheral_name`参数，或者传递了None，`source_peripheral_name`不会被清除
- 这导致即使选中了非继承的外设，位域图仍然显示继承信息
- 在`paintEvent`方法中（第70-91行），如果设置了`source_peripheral_name`，会显示继承信息

### 相关文件
- `svd_tool/ui/widgets/bit_field_widget.py` - 位域图组件
- `svd_tool/ui/managers/visualization_manager.py` - 可视化管理器

## 修复方案

### 修改1：修改set_register方法
**文件**: `svd_tool/ui/widgets/bit_field_widget.py`
**位置**: `set_register`方法（第29-50行）

**修改内容**：
在方法开始时，如果没有传递`source_peripheral_name`参数，或者传递了None，应该清除`source_peripheral_name`。

## 修复效果

### 修复前
- 选中非继承的外设时，位域图仍然显示继承信息
- 位域图显示混乱

### 修复后
- 选中非继承的外设时，位域图不显示继承信息
- 位域图显示清晰
