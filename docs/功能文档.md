## 一、项目概述

SVDEditor 是一款基于 Python/PyQt6 开发的 **CMSIS-SVD 文件解析/编辑/可视化工具**，面向芯片工程师和嵌入式开发者，提供图形化界面编辑 SVD（System View Description）文件，支持外设管理、寄存器编辑、位域可视化、中断管理等功能。

### 技术栈
| 技术 | 版本要求 | 用途 |
|------|---------|------|
| Python | 3.10+ | 运行环境 |
| PyQt6 | 6.5.0+ | GUI 框架 |
| xml.dom.minidom | 标准库 | XML 解析 |
| xml.etree.ElementTree | 标准库 | XML 生成 |
| argparse | 标准库 | CLI 接口 |

### 架构特点
- **组件化架构**：主窗口逻辑拆分为独立组件（StateManager, LayoutManager, PeripheralManager）
- **协调器模式**：Coordinator 组件注册与事件分发，解耦各模块
- **命令模式**：支持无限级撤销/重做（CommandHistory + Command）
- **国际化**：内置 i18n 管理，支持中英文切换

### 2.2 数据模型

| 模型 | 属性 | 说明 |
|------|------|------|
| **DeviceInfo** | name, version, description, vendor, copyright, author, license, cpu, addressUnitBits, width, size, resetValue, resetMask, svdVersion | 设备顶层信息 |
| **CPUInfo** | name, revision, endian, mpuPresent, fpuPresent, nvicPrioBits, vendorSystickConfig | CPU 配置信息 |
| **Peripheral** | name, baseAddress, description, displayName, groupName, derivedFrom, addressBlock, registers, clusters, interrupts | 外设定义 |
| **Cluster** | name, description, addressOffset, size, access, resetValue, resetMask, dim, dimIncrement, dimIndex, registers, clusters(嵌套) | 寄存器簇 |
| **Register** | name, offset, description, displayName, size, access, resetValue, resetMask, fields, derivedFrom | 寄存器 |
| **Field** | name, description, displayName, bitOffset, bitWidth, access, resetValue, enumeratedValues | 位域 |
| **Interrupt** | name, value, description, peripheral, peripherals | 中断 |

## 五、CLI 命令行接口

| 子命令 | 状态 | 说明 |
|--------|------|------|
| `validate` | ✅ 已实现 | CMSIS-SVD Schema 完整验证 |
| `export` | ✅ 已实现 | 导出为 CSV/Markdown/HTML |
| `generate` | ✅ 已实现 | 重新生成 SVD XML |
| `diff` | ✅ 已实现 | 比较两个 SVD 文件差异 |
| `info` | ✅ 已实现 | 显示 SVD 文件基本信息和统计 |
| JSON 输出 (`--json`) | ✅ 已实现 | 所有命令支持 JSON 格式输出 |
| 严格模式 (`--strict`) | ✅ 已实现 | 有错误/警告/差异时返回非零退出码 |
| 详细日志 (`-v`) | ✅ 已实现 | 开启 DEBUG 级别日志 |

### CLI 用法示例
```bash
python -m svd_tool.cli validate chip.svd
python -m svd_tool.cli export chip.svd --format markdown -o registers.md
python -m svd_tool.cli diff chip_v1.svd chip_v2.svd
python -m svd_tool.cli generate chip.svd -o output.svd
python -m svd_tool.cli info chip.svd --json
python run.py validate chip.svd    # 通过 run.py 入口
python run.py                      # 无参数启动 GUI
```
