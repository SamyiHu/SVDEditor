# 构建版本说明

## 两种构建模式

### 1. 单文件版本 (Single-file)
- **输出**: 单个 `.exe` 可执行文件
- **特点**: 
  - 所有依赖库、资源文件都打包在一个 `.exe` 文件中
  - 启动时会自动解压到临时目录运行
  - 文件大小较大（包含所有依赖）
  - 分发简单，只需一个文件
- **适用场景**:
  - 普通用户使用
  - 快速分发
  - 不需要查看内部文件的情况

### 2. 便携目录版本 (Portable directory)
- **输出**: 一个目录，包含：
  - `SVDEditor_XXbit.exe` (主程序)
  - 各种 `.dll` 文件 (依赖库)
  - `config.py` 等资源文件
- **特点**:
  - 文件结构清晰可见
  - 可以单独替换或修改某个文件
  - 目录整体可以移动（便携）
  - 启动速度可能稍快（无需解压）
- **适用场景**:
  - 开发者调试
  - 需要自定义配置
  - 需要查看依赖文件

## 架构选择 (32位 vs 64位)

### 32位版本
- 兼容性更好，可以在32位和64位Windows上运行
- 内存使用限制较多（最大2-4GB）
- 适合老旧系统

### 64位版本
- 只能在64位Windows上运行
- 可以使用更多内存
- 性能可能更好
- 适合现代系统

## 如何选择

| 需求 | 推荐选择 |
|------|----------|
| 普通用户，简单使用 | 单文件版本 + 对应系统架构 |
| 需要在不同电脑间移动 | 便携目录版本 |
| 需要调试或查看内部文件 | 便携目录版本 |
| 不确定目标系统架构 | 32位版本（兼容性更好） |
| 现代64位系统，需要更好性能 | 64位版本 |

## 构建脚本说明

### build_professional_fixed.py (专业构建脚本)
- 推荐使用
- 解决杀毒软件误报问题
- 目录结构更整洁
- 支持架构选择和版本选择

### build_windows.py (基础构建脚本)
- 简单直接
- 传统PyInstaller方式
- 同样支持架构选择

## 使用步骤

1. 运行 `build.bat` 或直接进入 `build_tools` 目录
2. 选择构建脚本（专业版或基础版）
3. 选择目标架构（32位或64位）
4. 选择构建模式（单文件或目录）
5. 等待构建完成

构建完成后，文件会输出到：
- 专业构建：`release/` 目录
- 基础构建：`dist/` 目录
